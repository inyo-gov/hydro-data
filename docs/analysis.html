summarise_dataset <- function(path, date_col = "date", staid_col = "staid") {
  if (!file.exists(path)) {
    return(tibble(
      File = basename(path),
      Rows = NA_integer_,
      Columns = NA_integer_,
      Unique_staids = NA_integer_,
      Date_start = NA,
      Date_end = NA
    ))
  }

  df <- read_csv(path, show_col_types = FALSE)
  has_date <- date_col %in% names(df)
  has_staid <- staid_col %in% names(df)

  date_vals <- if (has_date) ymd(as.character(df[[date_col]])) else as.Date(NA)
  staid_vals <- if (has_staid) df[[staid_col]] else character()

  tibble(
    File = basename(path),
    Rows = nrow(df),
    Columns = ncol(df),
    Unique_staids = if (has_staid) dplyr::n_distinct(staid_vals) else NA_integer_,
    Date_start = if (has_date) min(date_vals, na.rm = TRUE) else as.Date(NA),
    Date_end = if (has_date) max(date_vals, na.rm = TRUE) else as.Date(NA)
  )
}

Pastas pilot (groundwater time series modeling)

Pastas is a Python package for time-series modeling of groundwater heads with stress responses (recharge, pumping, trends). The typical workflow is:

  1. Load head observations as a pandas.Series indexed by date.
  2. Load stress time series (e.g., precipitation, pumping) as pandas.Series.
  3. Create a pastas.Model using head observations.
  4. Add stress models (e.g., recharge response).
  5. Solve and evaluate the model, then plot results.

Reference: Pastas modeling workflow (https://pastas.readthedocs.io/stable/userguide/pastas_modeling.html).

Main Pastas classes and methods

Core building blocks typically used in Pastas workflows:

  • ps.Model — container for the head series and fitted model.
  • ps.StressModel — links a stress series (e.g., pumping) to head response.
  • ps.RechargeModel — convenience stress model for precipitation/evaporation-derived recharge.
  • Response functions: ps.Gamma, ps.Exponential (shape the lagged response).
  • Solvers: ps.LeastSquares (default solver used by ml.solve()).
  • Common model methods: ml.add_stressmodel(), ml.solve(), ml.plot(), ml.plots.results().

Demo interpretation (fit metrics)

Once the model is solved, Pastas reports a fit summary with metrics such as:

  • R2: higher values indicate the model explains more variance in head observations.
  • RMSE: lower values indicate smaller average errors (same units as head data).
  • AICc/BIC: lower values indicate a better balance of fit and complexity; use for model comparison.
  • nobs: number of head observations used in the calibration period.
  • nfev: number of solver evaluations (useful for troubleshooting convergence).

For the demo series (T107 + Laws pumping stress), interpret results as:

  • If R2 is low (<0.3), the stress model likely misses dominant drivers (e.g., recharge).
  • If RMSE is large relative to seasonal head amplitude, the stress model is under-specified.
  • Use AICc/BIC to compare alternate stress configurations (pumping-only vs pumping+recharge).

Notes for integrating Pastas

  • Pastas requires Python and pastas installed in the Quarto environment.
  • Stress inputs should be daily or monthly time series aligned to the observation timeline.
  • For pumping impacts, monthly totals from totals_means_thru_sep_2025.csv can be converted to a stress series per wellfield or site.

Other analysis ideas (short list)

  • Trend detection (Mann-Kendall or Theil-Sen) on long-term depth-to-water series.
  • Seasonal decomposition of well hydrographs (STL) with WY/RY comparisons.
  • Correlation between pumping totals and nearby well responses by wellfield.
  • Spatial summary of monitoring point density by township or latitude band.
  • WY2025 anomalies vs historical percentiles for key indicator wells.

# Pastas pilot (groundwater time series modeling)

Pastas is a Python package for time-series modeling of groundwater heads with stress responses (recharge, pumping, trends). The typical workflow is:

1. Load head observations as a `pandas.Series` indexed by date.
2. Load stress time series (e.g., precipitation, pumping) as `pandas.Series`.
3. Create a `pastas.Model` using head observations.
4. Add stress models (e.g., recharge response).
5. Solve and evaluate the model, then plot results.

Reference: Pastas modeling workflow (https://pastas.readthedocs.io/stable/userguide/pastas_modeling.html).

## Main Pastas classes and methods

Core building blocks typically used in Pastas workflows:

- `ps.Model` — container for the head series and fitted model.
- `ps.StressModel` — links a stress series (e.g., pumping) to head response.
- `ps.RechargeModel` — convenience stress model for precipitation/evaporation-derived recharge.
- Response functions: `ps.Gamma`, `ps.Exponential` (shape the lagged response).
- Solvers: `ps.LeastSquares` (default solver used by `ml.solve()`).
- Common model methods: `ml.add_stressmodel()`, `ml.solve()`, `ml.plot()`, `ml.plots.results()`.
